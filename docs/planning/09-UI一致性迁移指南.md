# 09 - UI 一致性迁移指南

## 文档概述

本文档提供从原始 AionUi 项目到 Tuari 重写项目的 UI 设计系统完整迁移策略，确保视觉一致性和用户体验的连续性。

**目标**：
- 保持与原项目完全一致的视觉风格
- 迁移完整的设计令牌系统（CSS 变量）
- 复用 UnoCSS 配置和原子化 CSS 策略
- 迁移 Arco Design 组件定制
- 建立可维护的主题系统

---

## 1. 设计系统架构

### 1.1 原项目设计系统概览

原项目采用三层设计系统架构：

```
┌─────────────────────────────────────────┐
│  Layer 3: 组件层                         │
│  - Arco Design 组件库                    │
│  - 自定义 Base 组件                      │
│  - arco-override.css (234 行)           │
└─────────────────────────────────────────┘
              ↓ 使用
┌─────────────────────────────────────────┐
│  Layer 2: 原子化 CSS 层                  │
│  - UnoCSS 配置                           │
│  - 语义化颜色映射                        │
│  - 工具类生成                            │
└─────────────────────────────────────────┘
              ↓ 引用
┌─────────────────────────────────────────┐
│  Layer 1: 设计令牌层                     │
│  - CSS 变量 (base.css: 943 行)          │
│  - 颜色方案 (default.css: 142 行)       │
│  - TypeScript 类型 (colors.ts)          │
└─────────────────────────────────────────┘
```

### 1.2 核心设计原则

1. **CSS 变量优先**：所有颜色、间距、圆角等通过 CSS 变量定义
2. **明暗模式支持**：通过 `[data-theme="dark"]` 选择器切换
3. **语义化命名**：`--color-bg-0` 而非 `--white`
4. **原子化 CSS**：UnoCSS 生成工具类，避免重复样式
5. **组件定制**：通过 CSS 变量覆盖 Arco Design 默认样式

export const getCSSVar = (varName: string): string => {
  const name = varName.startsWith('--') ? varName : `--${varName}`;
  return getComputedStyle(document.documentElement).getPropertyValue(name).trim();
};

## 7. 主题切换实现

### 7.1 主题 Hook

创建 `src/shared/hooks/useTheme.ts`：

```typescript
import { useEffect, useState } from 'react';
import type { ThemeMode } from '@/shared/types/theme';

export function useTheme() {
  const [theme, setThemeState] = useState<ThemeMode>(() => {
    // 从 localStorage 读取
    const saved = localStorage.getItem('theme') as ThemeMode;
    return saved || 'auto';
  });

  useEffect(() => {
    const root = document.documentElement;

    // 确定实际主题
    let actualTheme: 'light' | 'dark';
    if (theme === 'auto') {
      actualTheme = window.matchMedia('(prefers-color-scheme: dark)').matches
        ? 'dark'
        : 'light';
    } else {
      actualTheme = theme;
    }

    // 应用主题
    root.setAttribute('data-theme', actualTheme);

    // 保存到 localStorage
    localStorage.setItem('theme', theme);
  }, [theme]);

  // 监听系统主题变化
  useEffect(() => {
    if (theme !== 'auto') return;

    const mediaQuery = window.matchMedia('(prefers-color-scheme: dark)');
    const handler = (e: MediaQueryListEvent) => {
      const root = document.documentElement;
      root.setAttribute('data-theme', e.matches ? 'dark' : 'light');
    };

    mediaQuery.addEventListener('change', handler);
    return () => mediaQuery.removeEventListener('change', handler);
  }, [theme]);

  return {
    theme,
    setTheme: setThemeState,
  };
}
```

## 10. 常见问题与解决方案

### 10.1 颜色不一致

**问题**：某些组件颜色与原项目不匹配

**排查步骤**：
1. 检查是否使用了硬编码颜色值
2. 验证 CSS 变量是否正确定义
3. 检查 UnoCSS 配置是否正确映射
4. 确认明暗模式切换是否生效

**解决方案**：
```typescript
// ❌ 错误：硬编码颜色
<div style={{ backgroundColor: '#F7F8FA' }}>

// ✅ 正确：使用 CSS 变量
<div className="bg-bg-1">

// ✅ 正确：内联样式使用 CSS 变量
<div style={{ backgroundColor: 'var(--color-bg-1)' }}>
```

### 10.2 间距不一致

**问题**：组件间距与原项目不同

**排查步骤**：
1. 检查是否使用了固定像素值
2. 验证 UnoCSS 间距配置
3. 检查是否遗漏了某些边距/内边距

**解决方案**：
```typescript
// ❌ 错误：固定像素值
<div className="p-16px">

// ✅ 正确：使用语义化间距
<div className="p-4"> {/* 16px */}

// ✅ 正确：使用 CSS 变量
<div style={{ padding: 'var(--spacing-4)' }}>
```

### 10.3 圆角不一致

**问题**：组件圆角大小不匹配

**解决方案**：
```typescript
// ❌ 错误：任意圆角值
<div className="rounded-10px">

// ✅ 正确：使用标准圆角
<div className="rounded-8px">  {/* --radius-medium */}
<div className="rounded-12px"> {/* --radius-large */}
```

### 10.4 Arco Design 组件样式覆盖失败

**问题**：Arco Design 组件样式无法覆盖

**原因**：CSS 优先级不足或加载顺序错误

**解决方案**：
```typescript
// 确保 arco-override.css 在 Arco Design 样式之后加载
// vite.config.ts
import { defineConfig } from 'vite';

export default defineConfig({
  css: {
    preprocessorOptions: {
      less: {
        modifyVars: {
          // Arco Design 主题变量
        },
      },
    },
  },
});

// main.tsx
import '@arco-design/web-react/dist/css/arco.css';
import './styles/arco-override.css'; // 必须在 Arco 样式之后
```

### 10.5 明暗模式切换不生效

**问题**：切换主题后颜色没有变化

**排查步骤**：
1. 检查 `data-theme` 属性是否正确设置在 `<html>` 元素上
2. 验证 CSS 变量是否在 `[data-theme="dark"]` 选择器中重新定义
3. 检查组件是否使用了硬编码颜色

**解决方案**：
```typescript
// useTheme.ts
export function useTheme() {
  const setTheme = (theme: ThemeMode) => {
    // 确保设置在 html 元素上
    document.documentElement.setAttribute('data-theme', theme);
    localStorage.setItem('theme', theme);
  };

  return { theme, setTheme };
}
```

4. **系统化验证**：通过检查清单和视觉测试确保完全一致

- `src/styles/themes/base.css` - 核心 CSS 变量定义

- `uno.config.ts` - UnoCSS 配置
- `src/styles/arco-override.css` - Arco Design 定制
