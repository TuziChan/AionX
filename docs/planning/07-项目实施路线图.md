## 1. 项目背景与目标

### 1.1 现状分析

**原项目（AionUi - Electron）**：
- 桌面框架：Electron 37
- 前端：React 19 + TypeScript 5.8 + Arco Design 2 + UnoCSS
- 后端：Node.js 22-24
- 数据库：SQLite (better-sqlite3)
- WebUI：Express 5 + WebSocket
- 打包体积：~150MB+
- 内存占用：高（Chromium进程）
- 代码行数：23,636 LOC
- 文件数量：626个TS/TSX文件

**核心功能**：
- 5个AI Agent（ACP/Claude、Gemini、Codex、Nanobot、OpenClaw）
- 多对话管理、流式响应、思考过程展示
- 工作空间系统、文件预览、代码编辑
- 10+设置页面、主题系统、6种语言国际化
- WebUI远程访问（JWT认证）
- 多平台通道（Telegram、飞书、钉钉）
- 定时任务系统、MCP集成、扩展系统

**现有问题**：
1. **代码重复率高**：5个Agent各有独立Chat/SendBox组件，重复率35-40%
2. **Context样板代码**：每个状态域需要Context → Provider → useXxx三件套
3. **深层嵌套**：最大嵌套深度6层
4. **职责不清**：common/shared/types目录职责重叠

### 1.2 迁移目标

**目标项目（AionX - Tauri 2.x）**：
- 桌面框架：Tauri 2.x
- 前端：React 19 + TypeScript 5.8 + Arco Design 2 + UnoCSS（保持一致）
- 后端：Rust（完全重写）
- 数据库：SQLite (sqlx)
- WebUI：Axum + WebSocket
- 打包体积：~10-20MB（减少85%+）
- 内存占用：低（系统WebView）
- 预期代码量：~55-60%（减少40-45%）

**核心改进**：
1. ✅ 统一Agent聊天架构 — 消除80%+重复代码
2. ✅ Zustand替代Context — 消除样板代码，性能提升
3. ✅ 扁平化目录结构 — 最大嵌套深度降至4层
4. ✅ 功能模块化 — features/目录按领域组织
5. ✅ Rust后端 — 性能、安全性、体积全面提升
6. ✅ 完全复刻UI — 界面和交互体验100%一致

## 4. 实施路线图

### 4.1 总体规划

项目分为 **6个阶段**，采用增量交付模式，每个阶段都能产出可验证的成果。

| 阶段 | 名称 | 工期 | 核心目标 | 交付物 |
|------|------|------|---------|--------|
| 阶段0 | 项目初始化 | 3-5天 | 搭建基础架构 | 可运行的空白应用 |
| 阶段1 | 基础设施层 | 5-7天 | 数据库、配置、事件系统 | 基础服务可用 |
| 阶段2 | 核心聊天功能 | 10-14天 | 单Agent聊天完整流程 | ACP聊天可用 |
| 阶段3 | 多Agent支持 | 7-10天 | 5个Agent全部接入 | 所有Agent可用 |
| 阶段4 | 高级功能 | 10-14天 | WebUI、通道、MCP、Cron | 完整功能集 |
| 阶段5 | 优化与发布 | 5-7天 | 性能优化、测试、打包 | 正式版本 |

**总工期**：40-57天（约6-8周）

## 8. 开发规范

### 8.1 Git工作流

- 使用 `main` 分支作为主分支
- 功能开发使用 `feature/xxx` 分支
- Bug修复使用 `fix/xxx` 分支
- 每个阶段完成后合并到 `main`
- 使用语义化版本号（Semantic Versioning）

### 8.2 代码规范

**Rust代码**：
- 使用 `rustfmt` 格式化
- 使用 `clippy` 检查
- 遵循 Rust API Guidelines
- 所有公开API必须有文档注释

**TypeScript代码**：
- 使用 ESLint + Prettier
- 严格模式（`strict: true`）
- 禁止使用 `any`
- 组件必须有类型定义

### 8.3 提交规范

使用 Conventional Commits：
- `feat:` 新功能
- `fix:` Bug修复
- `docs:` 文档更新
- `style:` 代码格式
- `refactor:` 重构
- `test:` 测试
- `chore:` 构建/工具

### 8.4 测试规范

- 单元测试覆盖率 > 70%
- 关键功能必须有集成测试
- UI组件必须有快照测试
- 每个阶段完成后进行回归测试
