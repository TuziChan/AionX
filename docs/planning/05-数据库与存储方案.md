# 05 - 数据库与存储方案

## 1. 数据库技术选型

| 对比项 | 原方案 (better-sqlite3) | 新方案 (sqlx) |
|--------|------------------------|-------------------|
| 语言 | Node.js 原生绑定 | Rust 原生 |
| 模式 | 同步 API | 异步 API（tokio） |
| 连接池 | 无（单连接） | 内置连接池 |
| 性能 | 优秀 | 更优（无 FFI 开销） |
| 迁移 | 手动版本管理 | sqlx migrate 或手动 |
| 打包 | 需要 prebuild | bundled 特性，零依赖 |
| 类型安全 | 运行时 | 编译期（宏验证） |

**选择 sqlx 的理由**：
- 编译期 SQL 验证（`sqlx::query!` 宏）
- 异步优先，与 Tauri 2.x 完美集成
- 内置连接池和迁移系统
- 活跃维护，生态成熟

// src-tauri/src/db/repositories/chat.rs
use sqlx::SqlitePool;

pub struct ChatRepository {
    pool: SqlitePool,
}

impl ChatRepository {
    pub fn new(pool: SqlitePool) -> Self {
        Self { pool }
    }

    pub async fn create(&self, input: &CreateChat, user_id: Option<&str>) -> Result<Chat, AppError> {
        let id = uuid::Uuid::new_v4().to_string();
        let now = chrono::Utc::now().timestamp();

        sqlx::query(
            "INSERT INTO chats (id, user_id, name, agent_type, model, workspace_path, extra, created_at, updated_at)
             VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?)"
        )
        .bind(&id)
        .bind(user_id)
        .bind(&input.name)
        .bind(&input.agent_type)
        .bind(input.model.as_deref().unwrap_or(""))
        .bind(input.workspace_path.as_deref().unwrap_or(""))
        .bind(serde_json::to_string(&input.extra.clone().unwrap_or_default())?)
        .bind(now)
        .bind(now)
        .execute(&self.pool)
        .await?;

        self.get(&id).await
    }

    pub async fn get(&self, id: &str) -> Result<Chat, AppError> {
        sqlx::query_as::<_, Chat>(
            "SELECT id, name, agent_type, model, workspace_path, created_at, updated_at
             FROM chats WHERE id = ?"
        )
        .bind(id)
        .fetch_optional(&self.pool)
        .await?
        .ok_or_else(|| AppError::NotFound(format!("Chat {id}")))
    }

    pub async fn list(&self, params: &ListParams) -> Result<PaginatedResult<Chat>, AppError> {
        let offset = (params.page.unwrap_or(1) - 1) * params.page_size.unwrap_or(50);
        let limit = params.page_size.unwrap_or(50) as i64;

        let total: (i64,) = sqlx::query_as(
            "SELECT COUNT(*) FROM chats WHERE status != 'deleted'"
        )
        .fetch_one(&self.pool)
        .await?;

        let items = sqlx::query_as::<_, Chat>(
            "SELECT id, name, agent_type, model, workspace_path, created_at, updated_at
             FROM chats WHERE status != 'deleted'
             ORDER BY updated_at DESC LIMIT ? OFFSET ?"
        )
        .bind(limit)
        .bind(offset as i64)
        .fetch_all(&self.pool)
        .await?;

        Ok(PaginatedResult {
            items,
            total: total.0,
            page: params.page.unwrap_or(1),
            page_size: limit as u32
        })
    }

    pub async fn update(&self, id: &str, updates: &ChatUpdate) -> Result<Chat, AppError> {
        let now = chrono::Utc::now().timestamp();
        let mut query_builder = sqlx::QueryBuilder::new("UPDATE chats SET updated_at = ");
        query_builder.push_bind(now);

        if let Some(ref name) = updates.name {
            query_builder.push(", name = ").push_bind(name);
        }
        if let Some(ref model) = updates.model {
            query_builder.push(", model = ").push_bind(model);
        }
        // ... 其他字段

        query_builder.push(" WHERE id = ").push_bind(id);
        query_builder.build().execute(&self.pool).await?;

        self.get(id).await
    }

    pub async fn delete(&self, id: &str) -> Result<(), AppError> {
        sqlx::query("DELETE FROM chats WHERE id = ?")
            .bind(id)
            .execute(&self.pool)
            .await?;
        Ok(())
    }

    pub async fn get_by_workspace(&self, path: &str) -> Result<Vec<Chat>, AppError> {
        let items = sqlx::query_as::<_, Chat>(
            "SELECT id, name, agent_type, model, workspace_path, created_at, updated_at
             FROM chats WHERE workspace_path = ? ORDER BY updated_at DESC"
        )
        .bind(path)
        .fetch_all(&self.pool)
        .await?;
        Ok(items)
    }

## 5. 数据库连接池

```rust
// src-tauri/src/db/mod.rs
use sqlx::sqlite::{SqliteConnectOptions, SqlitePool, SqlitePoolOptions};
use std::path::Path;

pub async fn init_database(app_data_dir: &Path) -> Result<SqlitePool, AppError> {
    let db_path = app_data_dir.join("aionx.db");

    // 配置 SQLite 连接选项
    let connect_options = SqliteConnectOptions::new()
        .filename(&db_path)
        .create_if_missing(true)
        .journal_mode(sqlx::sqlite::SqliteJournalMode::Wal)
        .foreign_keys(true)
        .busy_timeout(std::time::Duration::from_secs(5))
        .synchronous(sqlx::sqlite::SqliteSynchronous::Normal);

    // 创建连接池
    let pool = SqlitePoolOptions::new()
        .max_connections(10)
        .connect_with(connect_options)
        .await?;

    // 运行迁移
    run_migrations(&pool).await?;

    Ok(pool)
}
```

## 6. 迁移系统

```rust
// src-tauri/src/db/migration.rs
use sqlx::SqlitePool;

struct Migration {
    version: u32,
    description: &'static str,
    sql: &'static str,
}

const MIGRATIONS: &[Migration] = &[
    Migration {
        version: 1,
        description: "Initial schema",
        sql: include_str!("../../migrations/001_initial.sql"),
    },
    Migration {
        version: 2,
        description: "Add cron jobs",
        sql: include_str!("../../migrations/002_cron_jobs.sql"),
    },
    Migration {
        version: 3,
        description: "Add MCP servers",
        sql: include_str!("../../migrations/003_mcp_servers.sql"),
    },
    Migration {
        version: 4,
        description: "Add extensions",
        sql: include_str!("../../migrations/004_extensions.sql"),
    },
];

pub async fn run_migrations(pool: &SqlitePool) -> Result<(), AppError> {
    // 创建迁移跟踪表
    sqlx::query(
        "CREATE TABLE IF NOT EXISTS _migrations (
            version INTEGER PRIMARY KEY,
            description TEXT NOT NULL,
            applied_at INTEGER NOT NULL DEFAULT (unixepoch())
        );"
    )
    .execute(pool)
    .await?;

    let current_version: (i64,) = sqlx::query_as(
        "SELECT COALESCE(MAX(version), 0) FROM _migrations"
    )
    .fetch_one(pool)
    .await?;

    for migration in MIGRATIONS {
        if migration.version > current_version.0 as u32 {
            tracing::info!("Running migration v{}: {}", migration.version, migration.description);

            // 执行迁移SQL
            sqlx::raw_sql(migration.sql).execute(pool).await?;

            // 记录迁移
            sqlx::query(
                "INSERT INTO _migrations (version, description) VALUES (?, ?)"
            )
            .bind(migration.version as i64)
            .bind(migration.description)
            .execute(pool)
            .await?;
        }
    }

    Ok(())
}
```

## 8. 数据迁移策略

从原 Electron 版本迁移数据到 Tauri 版本：

1. 数据库文件格式兼容（都是 SQLite），可直接复制
2. 提供首次启动时的数据迁移向导
3. 自动检测原 Electron 版本的数据目录
4. 迁移内容：数据库文件 + 配置文件 + 扩展文件
