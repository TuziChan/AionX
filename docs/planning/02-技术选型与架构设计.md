# 02 - 技术选型与架构设计

## 1. 核心技术选型

### 1.1 桌面框架：Tauri 2.x

| 对比项 | Electron (现有) | Tauri 2.x (目标) |
|--------|----------------|-------------------|
| 后端语言 | Node.js | Rust |
| 打包体积 | ~150MB+ | ~10-20MB |
| 内存占用 | 高（Chromium 进程） | 低（系统 WebView） |
| 启动速度 | 较慢 | 快 |
| 安全模型 | 较宽松 | 严格权限系统 |
| 跨平台 | Win/Mac/Linux | Win/Mac/Linux + 移动端 |
| 插件生态 | npm 生态 | Tauri Plugins + crates.io |

选择 Tauri 2.x 的核心理由：
- 显著减小应用体积和内存占用
- Rust 后端提供更好的性能和安全性
- Tauri 2.x 的插件系统成熟，覆盖常见需求
- 权限系统更精细，适合安全敏感的 AI 应用

### 1.2 前端技术栈（保持 + 优化）

| 技术 | 选择 | 理由 |
|------|------|------|
| 框架 | React 19 | 保持一致，减少迁移成本 |
| 类型系统 | TypeScript 5.8+ | 保持一致 |
| UI 组件库 | Arco Design 2 | 保持一致，界面完全复刻 |
| 样式方案 | UnoCSS | 保持一致，原子化 CSS |
| 构建工具 | Vite 6 | Tauri 原生支持 |
| 状态管理 | Zustand | 替代 Context 三件套，减少样板代码 |
| 数据获取 | SWR | 保持一致 |
| 路由 | React Router 7 | 保持一致 |
| 国际化 | i18next | 保持一致 |

**关键变更：状态管理从 Context API 迁移到 Zustand**
- 消除 Context → Provider → useXxx 样板代码
- 更好的性能（避免不必要的重渲染）
- 更简洁的 API，代码量减少 50%+
- 内置 devtools 支持

## 3. 通信架构

### 3.0 端到端类型安全：tauri-specta

采用 **tauri-specta v2**（⭐680，Tauri 2 生态最活跃的类型安全方案）实现 Rust → TypeScript 的自动类型绑定：

**核心理念**：类型定义只在 Rust 端写一次（SSOT），前端 TS 绑定自动生成，编译期类型安全。

```rust
// Rust 端 — 只需加 #[specta::specta] 注解
use specta::Type;
use tauri_specta::{collect_commands, collect_events, Builder};

#[derive(Serialize, Deserialize, Type)]
pub struct Chat {
    pub id: String,
    pub title: String,
    pub messages: Vec<Message>,
}

#[tauri::command]
#[specta::specta]  // ← 一个注解搞定类型导出
async fn get_chat(id: String, state: State<'_, AppState>) -> Result<Chat, AppError> {
    state.chat_service.get(&id).await
}

// main.rs 中注册并导出绑定
let builder = Builder::<tauri::Wry>::new()
    .commands(collect_commands![
        get_chat, send_message, get_settings, /* ... */
    ])
    .events(collect_events![
        AgentStatusEvent, CronCompleteEvent, /* ... */
    ]);

#[cfg(debug_assertions)]
builder.export(specta_typescript::Typescript::default(), "../src/bindings.ts")?;
```

```typescript
// 前端 — 自动生成的 bindings.ts，全量类型安全
import { commands } from './bindings';

// IDE 自动补全 + 编译期类型检查，拼写错误直接报红
const chat = await commands.getChat('conv-1');
// chat 类型自动推导为 Chat
```

**备选方案对比**（选择 tauri-specta 的理由）：

| 方案 | Stars | 特点 | 不选的理由 |
|------|-------|------|-----------|
| **tauri-specta v2** ✅ | 680 | 注解式，与原生 API 完全兼容 | — |
| taurpc | ~56 | trait-based RPC，API 优雅 | Channel 流式支持不够成熟，与原生风格差异大 |
| rspc | 1366 | 类 tRPC 全栈框架 | 过重，引入独立路由层和中间件，杀鸡用牛刀 |
| 手动类型同步 | - | 无依赖 | 违反 SSOT，维护成本高，运行时才发现类型不匹配 |

### 3.2 后端 → 前端（Tauri Events，仅用于低频通知）

用于轻量广播通知（状态变更、Cron 完成等），**不用于流式数据**：

```rust
// Rust 端 — 带类型安全的事件定义
#[derive(Clone, Serialize, specta::Type, tauri_specta::Event)]
pub struct CronCompleteEvent {
    pub task_id: String,
    pub status: String,
}

// 发送事件
CronCompleteEvent { task_id, status }.emit(&app_handle)?;
```

```typescript
// 前端 — tauri-specta 自动生成类型安全的监听
import { events } from './bindings';
const unlisten = await events.cronCompleteEvent.listen((event) => {
  // event.payload 类型自动推导为 CronCompleteEvent
  console.log(event.payload.taskId, event.payload.status);
});
```

### 3.3 流式通信（Tauri Channel）

AI Agent 的流式响应通过 **Tauri Channel**（非 Event）实现，这是 Tauri 2.x 专为高吞吐低延迟流式数据优化的通信机制：

> **注意**：Tauri 官方明确指出 Event System 不适用于低延迟/高吞吐场景，应使用 Channel。

```
Agent Process → Rust Backend (tokio mpsc) → Tauri Channel → React UI
```

**Rust 端实现**：

```rust
use tauri::ipc::Channel;

#[derive(Clone, Serialize, specta::Type)]
#[serde(tag = "type")]
pub enum AgentEvent {
    MessageDelta { content: String },
    ToolCall { name: String, arguments: serde_json::Value },
    Progress { percent: f32, message: String },
    Complete { message: Message },
    Error { code: String, message: String },
}

#[tauri::command]
#[specta::specta]
async fn send_message(
    chat_id: String,
    content: String,
    on_event: Channel<AgentEvent>,
    state: State<'_, AppState>,
) -> Result<(), AppError> {
    let (tx, mut rx) = tokio::sync::mpsc::channel::<AgentEvent>(256);

    tokio::spawn(async move {
        state.agent_service.stream(&chat_id, &content, tx).await;
    });

    while let Some(event) = rx.recv().await {
        on_event.send(event)?;
    }

    Ok(())
}
```

**前端调用**：

```typescript
import { commands } from './bindings'; // tauri-specta 自动生成
import { Channel } from '@tauri-apps/api/core';

async function sendMessage(chatId: string, content: string) {
  const onEvent = new Channel<AgentEvent>();

  onEvent.onmessage = (event) => {
    switch (event.type) {
      case 'MessageDelta':
        appendToken(event.content);
        break;
      case 'ToolCall':
        handleToolCall(event);
        break;
      case 'Complete':
        finalizeMessage(event.message);
        break;
    }
  };

  await commands.sendMessage(chatId, content, onEvent);
}
```

**Channel vs Event 对比**：

| 维度 | Channel | Event |
|------|---------|-------|
| 适用场景 | 流式数据、高频推送 | 广播通知、低频事件 |
| 吞吐量 | 高（二进制序列化直传） | 低（JSON 字符串化） |
| 类型安全 | 编译期泛型检查 | 运行时弱类型 |
| 生命周期 | 与 Command 绑定，自动关闭 | 需手动 unlisten，易泄漏 |
| 背压控制 | 支持（配合 mpsc 容量限制） | 不支持 |

### 3.4 通信机制选择指南

| 场景 | 机制 | 理由 |
|------|------|------|
| AI 流式 token 输出 | **Channel** | 高频推送，低延迟 |
| 文件上传/下载进度 | **Channel** | 持续进度更新 |
| 对话/设置 CRUD | **Command** | 一次性请求响应 |
| Agent 启停控制 | **Command** | 操作性指令 |
| Cron 任务完成通知 | **Event** | 低频广播 |
| 设置/主题变更广播 | **Event** | 多窗口同步 |
| Channel 插件消息通知 | **Event** | 被动通知 |

## 5. 安全架构

### 5.1 Tauri 权限系统

```json
// tauri.conf.json 权限配置
{
  "app": {
    "security": {
      "capabilities": [
        {
          "identifier": "main-window",
          "windows": ["main"],
          "permissions": [
            "core:default",
            "shell:allow-spawn",
            "fs:allow-read",
            "fs:allow-write",
            "dialog:allow-open",
            "dialog:allow-save",
            "clipboard-manager:allow-write",
            "notification:allow-notify"
          ]
        }
      ]
    }
  }

```

### 5.2 安全策略
- 最小权限原则：前端只能通过 Command 访问后端功能
- 文件系统访问范围限制（Scope）
- 子进程执行白名单
- WebUI 认证强化（Argon2 替代 bcrypt）
- CSRF Token 保护
- 速率限制

## 6. 性能优化策略

| 优化点 | 方案 |
|--------|------|
| 启动速度 | Rust 编译优化 + 懒加载 |
| 内存占用 | 系统 WebView 替代 Chromium |
| 数据库性能 | sqlx WAL + 内置连接池 |
| AI 流式响应 | tokio channel + Tauri Event |
| 前端渲染 | React 19 并发特性 + 虚拟列表 |
| 打包体积 | Tauri 原生小体积 + 代码分割 |
