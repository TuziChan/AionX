# 02 - 技术选型与架构设计

## 1. 核心技术选型

### 1.1 桌面框架：Tauri 2.x

| 对比项 | Electron (现有) | Tauri 2.x (目标) |
|--------|----------------|-------------------|
| 后端语言 | Node.js | Rust |
| 打包体积 | ~150MB+ | ~10-20MB |
| 内存占用 | 高（Chromium 进程） | 低（系统 WebView） |
| 启动速度 | 较慢 | 快 |
| 安全模型 | 较宽松 | 严格权限系统 |
| 跨平台 | Win/Mac/Linux | Win/Mac/Linux + 移动端 |
| 插件生态 | npm 生态 | Tauri Plugins + crates.io |

选择 Tauri 2.x 的核心理由：
- 显著减小应用体积和内存占用
- Rust 后端提供更好的性能和安全性
- Tauri 2.x 的插件系统成熟，覆盖常见需求
- 权限系统更精细，适合安全敏感的 AI 应用

### 1.2 前端技术栈（保持 + 优化）

| 技术 | 选择 | 理由 |
|------|------|------|
| 框架 | React 19 | 保持一致，减少迁移成本 |
| 类型系统 | TypeScript 5.8+ | 保持一致 |
| UI 组件库 | Arco Design 2 | 保持一致，界面完全复刻 |
| 样式方案 | UnoCSS | 保持一致，原子化 CSS |
| 构建工具 | Vite 6 | Tauri 原生支持 |
| 状态管理 | Zustand | 替代 Context 三件套，减少样板代码 |
| 数据获取 | SWR | 保持一致 |
| 路由 | React Router 7 | 保持一致 |
| 国际化 | i18next | 保持一致 |

**关键变更：状态管理从 Context API 迁移到 Zustand**
- 消除 Context → Provider → useXxx 样板代码
- 更好的性能（避免不必要的重渲染）
- 更简洁的 API，代码量减少 50%+
- 内置 devtools 支持

## 3. 通信架构

### 3.1 前端 → 后端（Tauri Commands）

替代原有 29 个 IPC Bridge，统一为 Tauri Command 模式：

```rust
// Rust 端定义
#[tauri::command]
async fn get_conversation(id: String, state: State<'_, AppState>) -> Result<Conversation, AppError> {
    state.conversation_service.get(&id).await
}

// 前端调用
import { invoke } from '@tauri-apps/api/core';
const conversation = await invoke('get_conversation', { id: '...' });
```

### 3.2 后端 → 前端（Tauri Events）

替代原有 IPC 事件监听：

```rust
// Rust 端发送事件
app_handle.emit("agent-message", payload)?;
app_handle.emit("agent-status", status)?;

// 前端监听
import { listen } from '@tauri-apps/api/event';
const unlisten = await listen('agent-message', (event) => { ... });
```

### 3.3 流式通信

AI Agent 的流式响应通过 Tauri Event Channel 实现：

```
Agent Process → Rust Backend (tokio channel) → Tauri Event → React UI
```

## 5. 安全架构

### 5.1 Tauri 权限系统

```json
// tauri.conf.json 权限配置
{
  "app": {
    "security": {
      "capabilities": [
        {
          "identifier": "main-window",
          "windows": ["main"],
          "permissions": [
            "core:default",
            "shell:allow-spawn",
            "fs:allow-read",
            "fs:allow-write",
            "dialog:allow-open",
            "dialog:allow-save",
            "clipboard-manager:allow-write",
            "notification:allow-notify"
          ]
        }
      ]
    }
  }
}
```

### 5.2 安全策略
- 最小权限原则：前端只能通过 Command 访问后端功能
- 文件系统访问范围限制（Scope）
- 子进程执行白名单
- WebUI 认证强化（Argon2 替代 bcrypt）
- CSRF Token 保护
- 速率限制

## 6. 性能优化策略

| 优化点 | 方案 |
|--------|------|
| 启动速度 | Rust 编译优化 + 懒加载 |
| 内存占用 | 系统 WebView 替代 Chromium |
| 数据库性能 | sqlx WAL + 内置连接池 |
| AI 流式响应 | tokio channel + Tauri Event |
| 前端渲染 | React 19 并发特性 + 虚拟列表 |
| 打包体积 | Tauri 原生小体积 + 代码分割 |
